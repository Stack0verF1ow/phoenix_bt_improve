# Release Notes

## v0.3.0 — 局域网传输 + Flutter 移动端

### 重大变更

- **新增 Flutter 移动端** — Android 手机扫码连接电脑，双向传输文件
- **P2P 架构** — 任何设备（电脑或手机）都可充当服务端或客户端，角色可互换
- **局域网传输** — 基于 HTTP 的三步上传协议（prepare → upload → confirm），支持分块传输和断点续传

### 新增功能

- **扫码连接** — 电脑生成 PHX:// 二维码，手机扫码自动发现 IP 并连接
- **多网卡 IP 发现** — QR 码可包含多个局域网 IP（WiFi + 移动数据），连接方自动探测可达 IP
- **分块上传** — 4MB 分块，每块 CRC32 校验，支持暂停/继续
- **断点续传** — 上传中断后重新连接，跳过已完成的分块
- **共享文件** — 电脑端选择文件共享给手机，手机端自动刷新文件列表
- **接收模式** — 手机开启后生成二维码，让其他设备扫码传文件
- **自动做种** — 手机上传的文件可自动触发制种流程（制种 → 上传金凤 → uTorrent 做种）
- **BT 下载** — 移动端内置 BT 下载引擎，支持 .torrent 文件导入
- **文件管理** — 移动端查看、打开、分享、移动已下载文件
- **设备感知 UI** — 界面标签根据连接设备类型动态显示（电脑/手机）

### 体验优化

- 下载支持断点续传（HTTP Range）
- 上传/下载支持暂停和取消
- 文件夹按钮可打开系统文件管理器定位到下载目录
- 首页布局适配小屏设备（可滚动）
- 下载目录从应用私有存储迁移到外部存储（文件管理器可访问）

### Bug 修复

- 修复 CRC32 查找表数值错误导致上传校验失败
- 修复 429 限流：轮询端点（status/files）免除速率限制
- 修复下载取消不生效（改为关闭原生 HTTP 连接）
- 修复 OpenSSL Applink 崩溃（内网轮询改用 http.client）
- 修复 uTorrent peer_id 伪装

### 技术栈

`Python 3.12 + PySide6 + requests` | `Flutter 3.x + Dart + Dio + Provider`

## v0.2.0 — 全面重构，移除浏览器依赖

### 重大变更

- **移除 Selenium** — 不再需要 Edge/Chrome/Firefox 浏览器驱动
- **移除 Qt WebEngine** — 不再嵌入 Chromium 浏览器，安装包从 316MB 降至 **114MB**
- **HTTP 直连上传** — 修复 Cookie 处理逻辑后，ASP.NET 表单提交完全通过 `requests` 完成，不再需要真实浏览器环境
- **账号密码登录** — 直接输入金凤站点用户名密码即可，不再需要打开浏览器提取 Cookie

### 新增功能

- **uTorrent 路径跨盘自动检测** — 搜索所有盘符的 `Program Files` 和根目录下的 uTorrent/BitTorrent 安装
- **一键配置** — 首次运行向导自动检测 uTorrent 路径
- **设置自动保存** — 编辑字段后自动保存，不再需要手动点"保存"
- **设置页 UI 简化** — 分组清晰（站点登录 / Tracker / uTorrent 路径），移除不必要的内容

### 体验优化

- 做种按钮更大、蓝色主色调，操作路径更清晰
- 选择文件/文件夹按钮高亮，更容易发现
- 进度条更细更美观
- 日志框等宽字体，阅读更舒适
- "查看结果"按钮现在正确跳转到种子详情页

### 技术栈

`Python 3.12 + PySide6 + requests + BeautifulSoup4`
