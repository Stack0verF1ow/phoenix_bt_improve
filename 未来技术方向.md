# 未来技术方向

## 1. 纯 HTTP requests 替代 Selenium

**目标**：去掉浏览器依赖，用 requests 直接提交表单

**参考项目**：
- [nas-tools](https://github.com/NAStool/nas-tools) — Python + requests 多站自动上传
- [PT-Plugin-Plus](https://github.com/pt-plugins/PT-Plugin-Plus) — 浏览器插件，DOM 操作逻辑可参考

**技术要点**：
- ASP.NET WebForms 需提取 `__VIEWSTATE`、`__EVENTVALIDATION` 等隐藏字段
- multipart POST 上传种子文件
- 登录仍需 Selenium（用户交互），后续操作改用 requests + 缓存 cookies
- 跳过客户端 jQuery Validate，直接提交到服务器端

**优势**：上传秒完成，无浏览器启动开销
**风险**：服务器端可能有额外校验，需要逆向确认

## 2. Qt WebEngine 内置浏览器

**目标**：自包含，不依赖外部浏览器

**优势**：
- 跨平台一致（Windows/macOS/Linux）
- 无需 driver 管理
- 可直接操作页面 DOM

**代价**：
- 体积 +100~150MB
- 需重写自动化逻辑（QWebEngineView + QWebChannel）
- cookie 持久化需自己实现

## 3. macOS 适配

**方案 A**：GitHub Actions macOS runner 自动构建 .app
**方案 B**：Qt WebEngine 方案天然跨平台
**方案 C**：纯 requests 方案 + PyInstaller macOS 构建

**注意**：macOS 无法交叉编译，必须在 macOS 环境构建

## 4. 多站适配架构

**参考**：easy-upload、nas-tools 的站点适配器模式
- 定义统一的 `SiteAdapter` 接口
- 每个站点一个适配器模块
- 支持 NexusPHP、Gazelle、UNIT3D 等主流架构

## 5. 自动辅种（Cross-Seed）

**参考**：IYUUPlus
- 下载站点种子后，自动在其他站点查找相同资源
- 匹配 info_hash 或文件名/大小
- 自动下载并添加到 BT 客户端
