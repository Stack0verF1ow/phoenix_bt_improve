# 金凤本地做种助手 — 跨端传输 + 移动端 架构设计文档

## 一、需求概述

### 背景

当前已有 PC 端 Phoenix Helper（Python/PySide6），实现了完整的做种链路：制种 → HTTP 上传金凤 PT 站 → 下载站点官方种子 → 调用 uTorrent 做种。

现在需要开发移动端 App，并以 LAN 扫码传输作为桥梁，打通手机与 PC 之间的文件流转，最终以便捷的传输体验吸引用户使用 BT 站点。

### 功能需求

| 编号 | 功能 | 说明 |
|------|------|------|
| F1 | 手机扫码上传资源到电脑，自动做种 | 手机端扫码 → 选文件上传 → PC 接收 → 自动制种 → 上传金凤 → uTorrent 做种 |
| F2 | 电脑端文件传输到手机（顺带做种为选项） | PC 显示资源二维码 → 手机扫码下载。可选"顺带做种"（默认不勾，附小字鼓励文案） |
| F3 | 手机互传 | 一对一扫码，局域网 HTTP 直传 |
| F4 | 移动端 BT 下载 | Android 直接下载，iOS 下载 .torrent 后通过 share sheet 交由第三方 App |

### 产品策略

以局域网便捷传输为核心功能吸引用户 → 引导用户了解 BT 站点 → 鼓励用户做种。移动端不做种（避免用户嫌弃吃后台），做种能力集中在 PC 端。

### 三种传输场景

#### 场景 F1：手机 → PC（自动做种）

```
① PC 启动 LAN 服务，显示二维码（含 IP、端口、Token）
② 手机扫码 → 解析出 PC 地址 → 连接 PC
③ 手机选择文件 → POST /api/upload（multipart 上传）
④ PC 返回 upload_id，状态 "received"
⑤ 手机 POST /api/upload/<id>/confirm {auto_seed: true, title, category}
⑥ PC 后台线程开始做种流程：
   a. ResourceDraft.from_path(source_path)
   b. create_torrent() → 生成 .torrent
   c. PhoenixClient.upload_torrent() → 上传金凤
   d. PhoenixClient.download_final_torrent() → 下载官方种子
   e. UTorrentClient.open_torrent() → 启动 uTorrent 做种
⑦ 手机轮询 GET /api/upload/<id>/status 获取进度
```

#### 场景 F2：PC → 手机（下载，可选做种）

```
① PC 选择文件 → 生成带文件 ID 的二维码
② 手机扫码 → 解析出下载地址 → 开始下载
③ 手机端显示"顺带做种"复选框（默认不勾）
   文案："勾选后，PC 将同时制作 BT 种子并上传到金凤 PT 站进行做种，帮助资源长期保种。"
④ 如勾选 → 手机下载完成后 POST 确认 → PC 开始做种
```

#### 场景 F3：手机 ↔ 手机

```
① 手机 A 启动临时 HTTP 服务器 → 显示二维码
② 手机 B 扫码 → 连接手机 A
③ 手机 B 选择文件 → POST 上传 → 手机 A 接收
```

## 三、通信协议设计

### 3.1 二维码格式

采用 URL Scheme 格式，紧凑且可被任意扫码器识别：

```
PHX://v=1&t=pc&n=My+PC&h=192.168.1.100,10.0.0.5&p=18080&k=a3f8c2
```

| 字段 | 含义 | 示例 |
|------|------|------|
| `v` | 协议版本 | `1` |
| `t` | 设备类型 | `pc` 或 `phone` |
| `n` | 设备名（URL 编码） | `My+PC` |
| `h` | LAN IP 列表（逗号分隔） | `192.168.1.100,10.0.0.5` |
| `p` | 服务端口 | `18080` |
| `k` | Token 前 6 位（轻量认证） | `a3f8c2` |

### 3.2 HTTP API 端点（参考 LocalSend 三步模型）

基础路径：`http://<ip>:<port>`

认证方式：请求头 `X-Device-Token: <完整Token>` 或 `<Token前6位>`

响应格式：
- 成功：`{"status": "ok", ...}`（或特定状态码 + 业务字段）
- 错误：`{"status": "error", "message": "..."}`

#### 配对与会话

| 方法 | 路径 | 说明 | 请求体 | 响应 |
|------|------|------|--------|------|
| POST | `/api/register` | 扫码后用 token 前缀换取完整会话 | `{"token":"a3f8c2"}` | `{"status":"ok", "session":"完整Token", "device_name":"..."}` |
| GET | `/api/status` | PC 能力与状态 | — | `{"name","protocol_version","utorrent_available","phoenix_logged_in"}` |

#### 三步上传模型（参考 LocalSend prepare-upload → upload → cancel）

| 方法 | 路径 | 说明 | 请求体 | 响应 |
|------|------|------|--------|------|
| POST | `/api/prepare-upload` | **Step 1** — 告知文件名、大小、类型，获取 session + fileToken | `{"files":{"file0":{"name":"a.zip","size":1000,"type":"application/zip"}}}` | `{"sessionId":"uuid", "fileTokens":{"file0":"tok"}, "expires_in":600}` |
| POST | `/api/upload?sessionId=<sid>&fileId=<fid>&token=<tok>` | **Step 2** — 发送 raw bytes（无 multipart，纯二进制 body） | binary body | `{"status":"received", "fileId":"fid", "name":"a.zip", "size":1000}` |
| POST | `/api/confirm-seed` | **Step 3** — 确认完成，持久化到磁盘，可选触发做种 | `{"sessionId":"uuid", "auto_seed":bool, "title":"", "category":"1"}` | `{"status":"seeding"\|"idle", "uploads":[{"uploadId":"...","name":"..."}]}` |
| POST | `/api/cancel` | 取消会话（清理内存中的临时数据） | `{"sessionId":"uuid"}` | `{"status":"cancelled"}` |
| GET | `/api/upload/<sessionId>` | 轮询会话状态（已收文件、做种进度） | — | `{"sessionId","status","files_received","files_total","seed_status"}` |

**三步流程说明**：

```
Mobile                          PC
  │                              │
  │──── POST /api/register ──────┤  扫码配对
  │                              │
  │──── POST /api/prepare-upload ─┤  告知文件信息
  │         ← {sessionId, fileTokens}
  │                              │
  │──── POST /api/upload ────────┤  发送文件二进制（可并发多个）
  │   ?sessionId=xxx&fileId=f0   │
  │   &token=tok123              │
  │         ← {status:"received"}
  │                              │
  │──── POST /api/confirm-seed ──┤  确认完成，可选做种
  │   {sessionId, auto_seed:true}│
  │         ← {status:"seeding"} │
  │                              │
  │──── GET /api/upload/<sid> ───┤  轮询做种进度
  │         ← {seed_status:"done", detail_url:"..."}
```

#### 文件下载

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/files` | 列出 PC 共享目录中的文件/文件夹 |
| GET | `/api/files/download?path=<url编码路径>` | 下载指定文件（二进制流） |

## 四、PC 端改造方案（Phoenix Helper）

### 4.1 新增文件

```
src/phoenix_helper/
├── lan/
│   ├── __init__.py          # 空
│   ├── server.py            # HTTP 服务器（ThreadingHTTPServer）
│   ├── qr_generator.py      # QR 码生成
│   ├── file_store.py        # 接收文件管理
│   └── ip_utils.py          # LAN IP 发现
└── ui/
    ├── qr_display_widget.py  # QR 码展示组件
    └── lan_tab.py            # 第三标签页 "LAN传输"
```

### 4.2 修改文件

| 文件 | 改动 |
|------|------|
| `src/phoenix_helper/config.py` | 新增 `lan_port: int = 18080`、`device_name: str = ""`、`upload_receive_dir: Path` |
| `src/phoenix_helper/ui/main_window.py` | `_build_ui()` 新增 `_build_lan_tab()` 第三个 tab |
| `pyproject.toml` | 新增依赖 `qrcode[pil]>=7.4` |

### 4.3 新模块说明

#### `lan/server.py`

核心模块，管理 HTTP 服务器生命周期。关键类：

- **`LanRequestHandler(BaseHTTPRequestHandler)`** — 处理所有 LAN API 请求。共享状态（config、file_store、token）通过类属性注入。
- **`LanServer`** — 服务器管理类。`start(port)` 启动后台线程，`stop()` 关闭，`get_qr_content()` 生成二维码内容。

自动做种流程在 `lan_tab.py` 的 `SeedWorker(QThread)` 中实现，复用现有组件：

```
server 收到 POST /api/upload/<id>/confirm {auto_seed: true}
  → LanServer.on_seed_ready 回调
  → SeedWorker 后台线程:
     → ResourceDraft.from_path(source_path)
     → create_torrent(source, tracker, output)
     → PhoenixClient(config).upload_torrent(draft, torrent_path)
     → PhoenixClient().download_final_torrent(torrent_url, title, dir)
     → UTorrentClient().open_torrent(final_torrent, save_path=parent)
```

#### `lan/qr_generator.py`

- `build_qr_content()` — 构建 PHX:// URL
- `generate_qr_image()` — 使用 `qrcode[pil]` 生成 Pillow Image
- `qr_image_to_bytes()` — 转为 PNG bytes

#### `lan/file_store.py`

`FileStore` 类管理上传文件的磁盘存储：

- `create_upload(name, data)` → 保存到 `base_dir/<uuid>/原文件名` + `meta.json`
- `get_meta(upload_id)` / `update_meta(upload_id, **kwargs)` — 元数据读写
- `get_file_path(upload_id)` / `list_all()` / `remove(upload_id)`

#### `lan/ip_utils.py`

- `get_lan_ips()` → 使用 `socket.getaddrinfo` 获取所有非 127 开头的 IPv4 地址

### 4.4 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| HTTP 服务器 | stdlib `ThreadingHTTPServer` | 零额外依赖，LAN 场景够用 |
| QR 码生成 | `qrcode[pil]` | 成熟稳定，输出 Pillow Image 可转 Qt Pixmap |
| 自动做种 | 后台 SeedWorker QThread | 不阻塞 HTTP 服务器，复用现有做种代码 |
| IP 发现 | `socket.getaddrinfo` | 兼容性好，无需安装 netifaces |
| 文件传输 | HTTP multipart | 标准协议，手机端 Dio 原生支持 |

## 七、边界情况处理

### 7.1 校园网 NAT / 子网隔离

| 场景 | 方案 |
|------|------|
| 同一子网（同宿舍楼） | 直接 IP 连接，二维码含所有 LAN IP |
| 跨子网可路由 | IP 直连仍可用 |
| 跨子网不可路由 | 手动输入 IP 地址兜底 |
| 防火墙拦截 | 提示用户添加防火墙规则 |

### 7.2 大文件传输

使用 Dio 的 `onSendProgress` / `onReceiveProgress` 回调实时显示进度。后续可支持分块传输和断点续传。

### 7.3 并发连接

`ThreadingHTTPServer` 支持多线程并发处理。`FileStore` 使用 UUID 避免文件冲突。

## 十一、参考项目

以下开源项目与本方案的技术架构高度相关，在设计过程中作为参考。

### 11.1 LocalSend ⭐ 70k+

**GitHub**：[localsend/localsend](https://github.com/localsend/localsend)

全平台（Android/iOS/Windows/macOS/Linux）P2P 局域网文件传输，Flutter + Rust 实现。

| 特性 | 说明 |
|------|------|
| 设备发现 | UDP 多播（`224.0.0.167:53317`）+ HTTP 广播兜底 |
| 传输协议 | REST API over HTTPS（自签名 TLS），端口 53317 |
| API 流程 | `POST /prepare-upload` 协商元数据 → `POST /upload` 传文件 → `POST /cancel` 结束 |
| 安全 | 每个设备生成 SHA-256 fingerprint，防自发现；一次性 session token |
| 状态管理 | Refena（Provider + Redux 混合） |
| 性能层 | Rust（hyper、tokio、reqwest）处理 HTTP/TLS，防止 UI 线程阻塞 |
| 文件模型 | 统一的 `CrossFile` 模型，适配各平台文件系统 |

**对本项目的参考价值**：
- REST API 的三步传输模式（prepare-upload / upload / cancel）比我们当前的一步 POST 更健壮，后续可借鉴改进
- UDP 多播发现机制可替代二维码扫码做设备发现（但会增加复杂度，适合后续优化）
- 其 protocol 规范（[localsend-protocol](https://github.com/syntax-syndicate/localsend-protocol)）是局域网传输 API 设计的良好参考

### 11.2 QRPorter

**GitHub**：[manikandancode/qrporter](https://github.com/manikandancode/qrporter)

PySide6 + Flask 桌面应用，通过 QR 码在 PC 与手机浏览器之间传输文件。

| 特性 | 说明 |
|------|------|
| PC 端 | PySide6 GUI + Flask HTTP 服务器 |
| 手机端 | 任意浏览器（无需安装 App） |
| 传输模式 | PC→手机（扫码下载）、手机→PC（扫码上传） |
| 安全 | 一次性 session token、速率限制 |
| 安装 | `pip install qrporter` |

**对本项目的参考价值**：
- 确认了 PySide6 + HTTP 服务器 + QR 码的架构是成熟可行的
- 但它用 Flask 做后端（多一个依赖），我们直接用 stdlib `http.server` 更轻量
- 它没有手机 App，纯 Web 界面；我们加 Flutter App 是为了 BT 下载和更好的用户体验

### 11.3 Photon ⭐ 460+

**GitHub**：[abhi16180/photon](https://github.com/abhi16180/photon)

纯 Flutter 跨平台文件传输 App（Android/Windows/Linux/macOS/iOS），HTTP 直连。

| 特性 | 说明 |
|------|------|
| 传输方式 | 发送端启动 HTTP 服务器，接收端作为 HTTP 客户端下载 |
| 安全 | 每次会话生成加密 secret code 嵌入 URL 路径 |
| 特色 | 支持完整文件夹结构传输（保留子目录） |
| UI | Material You 设计 |

**对本项目的参考价值**：
- 验证了 Flutter HTTP 服务器 + 客户端模式的可行性
- 其 secret code 机制类似我们的 token 验证
- 文件夹结构保留功能可参考实现

### 11.4 share.py

**GitHub**：[swarooppatilx/share.py](https://github.com/swarooppatilx/share.py)

单文件 Python 脚本，`http.server` 共享文件 + `qrencode` 生成 QR 码。

**参考价值**：最简单的 stdlib HTTP 服务器 + QR 码方案，和我们 PC 端的实现思路一致，验证了零额外依赖路线的可行性。

### 11.5 总结对照

| 本方案设计点 | 参考项目 | 结论 |
|-------------|----------|------|
| PC 端 HTTP 服务器 + QR | QRPorter、share.py | ✅ 已验证可行 |
| Flutter HTTP 客户端/服务器 | Photon、LocalSend | ✅ 已验证可行 |
| REST API 传输协议 | LocalSend（最完整） | ✅ 可参考其 prepare-upload 三步模式 |
| QR 扫码配对连接 | 三者都支持 | ✅ 成熟方案 |
| **BT 站点做种集成** | **无竞品** | **本项目的核心差异点** |
